# AI Standards

> Guia de uso de IA, prompts e padroes de manutencao para manter o Cidadao Informa consistente no longo prazo.

## Objetivo

Esta pasta padroniza como pessoas e agentes de IA devem trabalhar no projeto. O foco e reduzir retrabalho, manter a identidade publica brasileira, preservar seguranca/LGPD e evoluir o produto sem quebrar fluxos existentes.

## Como Usar

Antes de pedir uma mudanca para uma IA ou iniciar uma tarefa manual:

1. Leia o documento relacionado ao tipo de tarefa.
2. Use um prompt da biblioteca e preencha os campos entre colchetes.
3. Execute os checks do checklist de manutencao.
4. Atualize a documentacao quando houver mudanca de fluxo, regra, API, dados ou decisao arquitetural.

## Principios Permanentes

- Produto publico deve ser simples, acessivel e rastreavel.
- IA apoia decisao, mas nao substitui revisao humana em fluxo administrativo.
- Dados pessoais devem ser minimizados e protegidos.
- Mudancas pequenas e verificadas valem mais que grandes refatoracoes sem teste.
- O padrao visual deve continuar alinhado a uma identidade publica brasileira: azul institucional, verde/amarelo como acentos e vermelho apenas para erro/risco.
